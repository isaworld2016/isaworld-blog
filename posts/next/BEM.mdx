## 0. 갈아탄 이유
아래의 이전 코드를 보자.

### before
```css showLineNumbers
.navigation.pagination {
  display: -webkit-box;
  display: flex;
}
.navigation.pagination > .nav-links{
  margin: 2px 0 0 0;
  display: -ms-flexbox;
  display: flex;
}
.navigation.pagination > .nav-links > .page-numbers {
  display: inline-block;
  margin: 0 8px;

  text-decoration: underline;
  text-decoration-thickness: 3px;
}

/**이하 생략 */
```

나중돼서 보면 보기싫을 것 같은 코드인데, 이유는!

종속적 포함 관계가 가시적으로 보이지 않기 때문일 것이다.

게다가 클래스명에서도 중복이 많아서 가로로 길~어진 것을 볼 수 있다.

<b>Sass</b>를 설치해보자.


## 1. Nextjs에 Sass 뿌리기

```
$npm install sass
```

global.css -> global.scss로 변경.


## 2. BEM 뿌리기
이걸로는 부족하다!

<b>[BEM](https://velog.io/@nemo/bem)</b>을 뿌리자.

> BEM이란 Block, Element, Modifier의 약자

- class명을 종속적으로 기술하는 게 아니라 블럭으로 구성.

- 여기서 블럭은 모듈/컴포넌트처럼 '독립적으로 존재하는 덩어리' 라고 생각하면 됨.

- "블럭" __ "엘리먼트" -- "유니크한 속성"


## 3. After
어떻게 바뀌었을까!

```css showLineNumbers
/**mixin을 활용하면 css의 중복된 정의를 줄일 수 있다. */
@mixin icon__a {
  display: -webkit-box;
  display: flex;
  margin: 0 0 0 20px;
  font-family: "Inter",sans-serif;
  font-size: 15px;
  font-weight: bold;
  cursor:pointer;
}
@mixin icon--rotate {
  transform:rotate(180deg);
}

.pagination{
  display: flex;
  padding: 20px 0 60px 0;
  position:relative;
  width: 100%;
  justify-content: center;
  -webkit-box-pack: center;

  &__wrap /**BEM 스타일로 클래스명을 달았더니 가독성이 좋아졌다. prefix가 달려있어서 그런가보다.*/
    display: flex;
    align-items: center;
    -webkit-box-pack: center;
    justify-content: center;
    height: 43px;
    border-width: 1px;
    border-style: solid;
    border-color: #000;
    border-radius: 6px;
    padding: 0 20px 0 20px;

    .icon__previous{
      display: flex;
      a{
        @include icon__a; /**mixins로 선언하고 include로 가져다가 쓸 수 있다. */
      }
      &--rotate-180{
        @include icon--rotate;
      }
    }
  }
}
/**이하 생략.. */
```
